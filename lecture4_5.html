Kali linux and VmWare

Kali Linux is a Debian-based distribution specifically tailored to the needs of penetration testing, digital forensics, and security auditing. It includes a large collection of tools and utilities for these purposes, such as Wireshark, Nmap, and Aircrack-ng. Additionally, Kali Linux is designed to be lightweight and portable, making it easy to carry on a USB drive or other portable storage device, and to run on a wide range of hardware, including laptops and ARM devices such as the Raspberry Pi.

One of the key features of Kali Linux is its rolling release model, which means that instead of having to download a new version of the operating system every few months or years, updates are delivered continuously. This ensures that users always have the latest security updates and tools at their disposal.

Some common use cases of Kali Linux are:

  • Penetration Testing
  • Ethical Hacking
  • Digital Forensics
  • Reverse Engineering
  • Wireless Network Auditing etc.

Using Wireshark

Wireshark is a packet sniffer and analysis tool. It captures network traffic from ethernet, Bluetooth, wireless (IEEE.802.11), token ring, and frame relay connections, among others, and stores that data for offline analysis.

Capturing data packets on Wireshark

  1. When you open Wireshark, you see a screen showing you a list of all the network connections you can monitor. You also have a capture filter field to only capture the network traffic you want to see.


  2. Click the first button on the toolbar, titled “Start capturing packets.”




  3. Once you have captured all the packets needed, stop the capturing and start analyzing each packet one after the other.




Implementing HoneyPot

To install honeypot in your Kali Linux machine, you need to download and install Pentbox which you can easily find it through Github repository. git clone https://github.com/technicaldata/pentbox-1.8.git. After downloading, you can simply run the pentbox by typing “./pentbox.rb” from pentbox-1.8 folder.





Once it is installed, let us start using the paintbox. Select the network tools and honeypot from the menu to install the honeypot. Go with the manual configuration to install it according to your preferences for a honeypot.





Now you can open the fake port as you wish and insert a fake message. You can also provide the ability to save the record and the record’s name. You can see the honeypot is enabled on the required port, and you can manually enable honeypots for other ports in a similar manner.
